1.2 OS Installations & Upgrades

🚀 Boot Methods

USB Boot

Most common method

• Fast installation

• Portable media

• UEFI support

Network Boot

PXE (Preboot Execution)

• Central deployment

• No physical media

• Automated process

Internal HDD

Partition-based boot

• Multiboot scenarios

• Recovery partitions

• Dual-boot setups

📦 Installation Types

Definition: Complete OS installation on a formatted drive

Pros: Fresh start, optimal performance, no legacy issues

Cons: Data loss, need to reinstall applications

When to use: Corrupted systems, major upgrades, new computers

Definition: Installing newer OS version over existing one

Pros: Preserves data and applications

Cons: Potential compatibility issues, slower performance

Requirements: Compatible hardware, supported upgrade path

Definition: Deploying pre-configured OS image

Tools: Windows Deployment Services (WDS), Ghost, Clonezilla

Benefits: Standardized configurations, rapid deployment

Use case: Corporate environments, multiple identical systems

⚠️ Important: Always backup critical data before any OS installation or upgrade process!

💾 Partitioning

GPT (GUID Partition Table)

• Modern standard

• UEFI requirement

• 128 partitions max

• >2TB drive support

MBR (Master Boot Record)

• Legacy standard

• BIOS compatibility

• 4 primary partitions

• 2TB drive limit

1.5 Command Line Tools

📁 Navigation Commands

C:\> cd Documents
C:\Users\Username\Documents> dir
Volume in drive C has no label.
Directory of C:\Users\Username\Documents
C:\Users\Username\Documents> cd ..
C:\Users\Username>

cd (Change Directory)

• cd.. (go up one level)

• cd\ (go to root)

• cd /d D: (change drive)

dir (Directory listing)

• dir /a (show hidden files)

• dir /s (include subdirectories)

• dir *.txt (filter by type)

🌐 Network Commands

C:\> ipconfig /all
Windows IP Configuration
Host Name: COMPUTER-01
IPv4 Address: 192.168.1.100
C:\> ping google.com
Pinging google.com [172.217.164.110] with 32 bytes:
Reply from 172.217.164.110: bytes=32 time=20ms TTL=116

ipconfig

• ipconfig /all (detailed info)

• ipconfig /release (release IP)

• ipconfig /renew (renew IP)

• ipconfig /flushdns (clear DNS cache)

ping

Test network connectivity

• ping -t (continuous ping)

• ping -4 (force IPv4)

tracert

Trace route to destination

Shows hop-by-hop path

Identifies network bottlenecks

netstat

• netstat -a (all connections)

• netstat -n (numerical addresses)

• netstat -b (show programs)

💾 Disk Management Commands

chkdsk

Check disk for errors

• chkdsk /f (fix errors)

• chkdsk /r (locate bad sectors)

diskpart

Advanced disk partitioning

• list disk, select disk

• create partition primary

sfc (System File Checker)

• sfc /scannow (scan system files)

• sfc /verifyonly (verify only)

⚠️ Caution: Some commands like diskpart and format can cause data loss. Always double-check your commands!

1.9 Linux Client/Desktop

📁 File Management Commands

user@linux:~$ ls -la
total 24
drwxr-xr-x 3 user user 4096 Oct 15 14:30 .
drwxr-xr-x 3 root root 4096 Oct 15 14:25 ..
-rw-r--r-- 1 user user 220 Oct 15 14:25 .bash_logout
user@linux:~$ pwd
/home/user

ls (list)

• ls -l (long format)

• ls -a (show hidden)

• ls -la (long + hidden)

pwd (print working directory)

Shows current location

Full path display

mv (move/rename)

mv file1 file2 (rename)

mv file /path/ (move)

cp (copy)

• cp file1 file2

• cp -r dir1 dir2 (recursive)

👤 Administrative Commands

su (switch user)

• su - (login shell)

• su username

Requires target user password

sudo (super user do)

Execute commands as another user

Uses current user password

Configurable permissions

user@linux:~$ chmod 755 script.sh
user@linux:~$ chown user:group file.txt

Permission Numbers:

  • 4: Read (r)
  • 2: Write (w)
  • 1: Execute (x)
  • 755: rwxr-xr-x (owner: rwx, group: r-x, others: r-x)
