4.1 Cloud Computing Concepts

Common Cloud Models

🔒 Private Cloud

Dedicated cloud infrastructure for a single organization. Offers maximum control and security but requires significant investment.

Use Case: Large enterprises with strict security requirements

🌐 Public Cloud

Cloud services offered over the public internet by providers like AWS, Azure, or Google Cloud. Cost-effective and scalable.

Use Case: Startups, small businesses, development environments

🔀 Hybrid Cloud

Combination of private and public clouds, allowing data and applications to move between them for flexibility.

Use Case: Organizations needing both security and scalability

👥 Community Cloud

Shared cloud infrastructure for organizations with common requirements (e.g., compliance, security).

Use Case: Healthcare networks, government agencies

Cloud Characteristics

📊 Shared Resources

Multiple users share the same physical infrastructure through virtualization

📈 Metered Utilization

Pay-per-use billing model based on actual resource consumption

⚡ Rapid Elasticity

Ability to quickly scale resources up or down based on demand

🔄 High Availability

Services remain accessible with minimal downtime through redundancy

☁️ File Synchronization

Automatic syncing of files across multiple devices and locations

Desktop Virtualization

🏢 VDI On-Premises

Virtual desktops hosted on local servers within the organization's data center. Provides control but requires infrastructure investment.

☁️ VDI in the Cloud

Virtual desktops hosted by cloud providers. Reduces infrastructure costs and provides scalability.

Security Requirements

🛡️ Isolation

VMs must be properly isolated from the host system and other VMs to prevent security breaches.

🔐 Access Control

Implement strong authentication and authorization for VM access and management.

📊 Monitoring

Continuous monitoring of VM activities and resource usage for security and performance.

💾 Backup & Recovery

Regular snapshots and backups to ensure data protection and quick recovery.
