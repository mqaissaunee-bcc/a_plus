Logical Security

Access Control Principles

‚öñÔ∏è Principle of Least Privilege

Users should have the minimum access rights necessary to perform their job functions

Implementation: Role-based access, need-to-know basis, regular access reviews

üìã Access Control Lists (ACLs)

Define which users or system processes can access specific resources

Components: Subject (user), Object (resource), Action (permission)

üîê Multifactor Authentication (MFA)

Something you know: Password, PIN

Something you have: Token, smart card

Something you are: Biometric data

MFA Implementation Methods

üìß Email-based MFA

Verification codes sent to registered email addresses

Pros: Easy to implement | Cons: Email account compromise risk

üé´ Hard Tokens

Physical devices generating time-based codes (RSA SecurID)

Pros: Highly secure | Cons: Cost, physical management

üì± Soft Tokens

Mobile apps generating authentication codes (Google Authenticator)

Pros: Convenient, cost-effective | Cons: Device dependency

üìû Voice/SMS

Codes delivered via phone call or text message

Pros: Universal access | Cons: SIM swapping attacks

Wireless Security Protocols

Encryption Protocols

üîí WPA2 (Wi-Fi Protected Access 2)

Encryption: AES-CCMP (Advanced Encryption Standard)

Key Management: 4-way handshake for key establishment

Security: Strong protection against most attacks

Status: Widely deployed, considered secure

üõ°Ô∏è WPA3 (Latest Standard)

Improvements: Protection against offline dictionary attacks

SAE: Simultaneous Authentication of Equals

Forward Secrecy: Past sessions remain secure if password is compromised

Adoption: Newer devices, gradually replacing WPA2

‚ö†Ô∏è TKIP (Temporal Key Integrity Protocol)

Legacy: Used with WPA (original)

Weaknesses: Vulnerable to various attacks

Replacement: Superseded by AES in WPA2

Status: Deprecated, avoid using

üîê AES (Advanced Encryption Standard)

Algorithm: Symmetric encryption with 128/192/256-bit keys

Performance: Fast and efficient

Security: Military-grade encryption

Usage: Standard for WPA2 and WPA3

Authentication Methods

üåê RADIUS (Remote Authentication Dial-In User Service)

Purpose: Centralized authentication, authorization, and accounting

Protocol: UDP-based client-server protocol

Enterprise Use: Large-scale wireless deployments

üîí TACACS+ (Terminal Access Controller Access-Control System Plus)

Cisco Protocol: Separates authentication, authorization, and accounting

Security: Encrypts entire packet payload

Flexibility: Granular control over user privileges

üé´ Kerberos

Tickets: Time-limited authentication tokens

KDC: Key Distribution Center manages authentication

Mutual Auth: Both client and server authenticate each other
